# Readyme 서비스 기획안

## 1. 서비스 개요

### 한 줄 소개

**이력서 템플릿을 만들고, 공유하고, 보상받는 플랫폼**

### 서비스 정의

Readyme는 노코드 빌더로 이력서 템플릿을 직접 제작하고, 이를 공유하거나 판매할 수 있는 플랫폼입니다. 다른 사람의 템플릿을 가져와 나만의 이력서를 완성하고, 완성된 이력서도 공유하거나 판매할 수 있습니다. 열람이 늘어날수록 작성자에게 보상이 돌아갑니다.

### 핵심 가치

| 사용자        | 가치                                                                     |
| ------------- | ------------------------------------------------------------------------ |
| 템플릿 제작자 | 노코드 빌더로 이력서 레이아웃을 만들어 공유/판매. 열람·구매 시 보상 적립 |
| 이력서 작성자 | 템플릿을 선택해 자신의 정보만 입력하면 이력서 완성. 공유/판매도 가능     |
| 열람자        | 다양한 템플릿과 이력서를 탐색하며 참고                                   |
| 피드백 제공자 | 이력서에 피드백을 남기며 커뮤니티 기여                                   |

---

## 2. 핵심 개념

### Template vs Resume

```
[react-rnd + tiptap 빌더] → 템플릿 제작 → 공유/판매 (구매자: 복제 후 수정 가능)
                               ↓
                         템플릿 선택 → 데이터 입력 → 이력서 완성 → 공유/판매 (구매자: 조회만 가능)
```

|           | Template (템플릿)                  | Resume (이력서)                         |
| --------- | ---------------------------------- | --------------------------------------- |
| 정의      | react-rnd + tiptap으로 만든 이력서 레이아웃 | 템플릿에 실제 데이터를 채운 결과물      |
| 제작 방식 | react-rnd로 블록 배치, tiptap으로 텍스트 편집 | 템플릿의 편집 가능 영역에 텍스트 입력 |
| 구매 시   | **복제하여 자유롭게 수정 가능**    | **조회만 가능 (읽기 전용)**             |
| 판매      | 무료/유료, 언제든 전환 가능        | 무료/유료, 언제든 전환 가능             |

## 3. 타겟 사용자

### 주요 타겟

- **취업 준비생**: 좋은 템플릿을 골라 자신의 정보만 채워 이력서 작성
- **디자인 감각 있는 사용자**: 노코드 빌더로 독창적인 템플릿을 만들어 판매
- **시니어/멘토**: 자신의 이력서를 공개하여 후배에게 참고 자료 제공, 피드백 제공

### 사용자 시나리오

**시나리오 1 — 취준생 A**

1. GitHub 계정으로 가입
2. 템플릿 마켓에서 마음에 드는 무료 템플릿 선택
3. 자신의 경력/학력/기술 데이터를 입력
4. 이력서 완성 → 무료로 공개 → 피드백을 받아 개선

**시나리오 2 — 디자이너 B**

1. 노코드 빌더에서 깔끔한 이력서 레이아웃 제작
2. 유료(3,000원)로 템플릿 게시
3. 취준생들이 템플릿 구매 → 복제하여 자기 데이터 입력
4. 구매 금액의 90%가 B에게 적립

**시나리오 3 — 시니어 개발자 C**

1. 인기 템플릿을 가져와 10년차 경력 이력서 완성
2. 유료(5,000원)로 이력서 게시
3. 취준생들이 참고용으로 구매 (조회만 가능, 수정 불가)
4. 조회수에 따른 추가 보상도 적립

## 4. 핵심 기능

### 4-1. 노코드 템플릿 빌더

- A4 고정 캔버스 기반 에디터
- react-rnd로 블록 요소를 자유롭게 드래그/리사이즈로 배치
- tiptap 리치 텍스트 에디터로 블록 내 콘텐츠 편집 (서식, 링크, 리스트 등)
- 텍스트 블록에 `editable` 속성 → 이력서 작성 시 편집 가능 영역 지정
- 자동 저장, 미리보기, 사용자 썸네일 업로드

### 4-2. 이력서 작성

- 템플릿 선택 → 편집 가능 영역에 tiptap 에디터로 데이터 입력
- 실시간 미리보기 (템플릿 레이아웃 + 입력 데이터)
- 자동 저장

### 4-3. 템플릿/이력서 게시 (무료/유료)

| 구분        | 무료 게시               | 유료 게시                         |
| ----------- | ----------------------- | --------------------------------- |
| 열람        | 누구나 열람             | 미리보기만 → 결제 후 접근         |
| 보상        | 조회수 기반 소액 포인트 | 구매 금액의 90% 적립              |
| 전환        | 언제든 유료로 전환 가능 | 언제든 무료로 전환 가능           |
| 기존 구매자 | -                       | 전환 후에도 기존 구매자 권한 유지 |

**구매 후 차이**:

- 템플릿 구매 → 복제하여 자유롭게 수정 가능 (내 소유 템플릿으로 생성)
- 이력서 구매 → 조회만 가능 (읽기 전용)

### 4-4. 탐색

- **템플릿 마켓**: 카테고리(개발자, 디자이너, 일반 등), 무료/유료, 인기순/최신순/평점순/사용순
- **이력서 탐색**: 직군, 기술 스택, 무료/유료 필터, 인기순/최신순/평점순

### 4-5. 열람 보상 시스템

```
열람/구매 → viewCount++ → 작성자 보상 적립 (balance +=)
```

- **무료 콘텐츠**: 열람 N회마다 소액 포인트 적립 (예: 10회당 100P)
- **유료 콘텐츠**: 구매 금액의 90% 즉시 적립
- 템플릿과 이력서 모두 동일한 보상 정책

**어뷰징 방지**:

- 본인 콘텐츠 조회는 viewCount 증가 안 함
- 비로그인 조회는 보상 대상 제외
- 동일 사용자 동일 콘텐츠 일일 중복 조회 제한

### 4-6. 피드백

- 공개 이력서에 직접 피드백 작성 (내용 + 별점)
- 특정 섹션(fieldKey) 지정 피드백 가능
- **비공개**: 피드백은 이력서 작성자(본인)만 열람 가능
- **글자수 제한**: 최대 2,000자 (리뷰보다 긴 상세 첨삭 용도)

### 4-7. 평가 (리뷰)

판매자, 템플릿, 이력서에 대해 누구나 평가를 남길 수 있습니다.

#### 평가 대상

| 대상   | 평가 자격        | 평가 항목                          |
| ------ | ---------------- | ---------------------------------- |
| 판매자 | 로그인 사용자    | 별점(1~5) + 한줄평 (최대 200자)    |
| 템플릿 | 로그인 사용자    | 별점(1~5) + 리뷰 텍스트 (최대 500자) |
| 이력서 | 로그인 사용자    | 별점(1~5) + 리뷰 텍스트 (최대 500자) |

#### 평가 정책

- **누구나 작성 가능**: 로그인한 사용자라면 리뷰 작성 가능
- **구매자 뱃지**: 유료 구매 이력이 있는 사용자의 리뷰에 "구매 인증" 뱃지 표시
- **1인 1리뷰**: 동일 대상에 대해 한 번만 작성 가능 (수정은 가능)
- **삭제 불가**: 작성자가 수정만 가능, 삭제는 신고 후 관리자 처리
- **평균 별점 노출**: 상세 페이지 및 목록에 평균 별점(avgRating)과 리뷰 수(reviewCount) 표시
- **판매자 평점**: 해당 판매자가 받은 모든 리뷰(판매자/템플릿/이력서) 별점의 평균으로 자동 산출

#### 정렬/필터

- 탐색 페이지에서 평점순 정렬 지원
- 최소 리뷰 수 필터 (예: 리뷰 5개 이상만 평점순 노출)

### 4-8. 알림

인앱 알림(DB 기반):

- "내 이력서에 피드백이 달렸습니다"
- "내 템플릿/이력서가 구매되었습니다"
- "보상 N포인트가 적립되었습니다"
- "내 템플릿/이력서에 리뷰가 등록되었습니다"

## 5. 페이지 구조 (사이트맵)

```
/                              홈 (서비스 소개)
├── /login                     로그인 (이메일 + GitHub)
├── /register                  회원가입
├── /templates                 템플릿 마켓플레이스
│   └── /templates/[id]        템플릿 상세/미리보기
├── /builder                   노코드 템플릿 빌더 (새 템플릿)
│   └── /builder/[id]          기존 템플릿 편집
├── /explore                   이력서 탐색
├── /r/[slug]                  이력서 열람 (공개 URL, 읽기 전용)
├── /resumes                   내 이력서 목록
│   ├── /resumes/new           새 이력서 작성 (템플릿 선택)
│   └── /resumes/[id]/edit     이력서 편집
├── /mypage                    마이페이지
│   ├── /mypage/templates      내 템플릿 관리
│   ├── /mypage/feedbacks      피드백 대시보드
│   ├── /mypage/reviews        내 리뷰 관리 (작성한/받은 리뷰)
│   └── /mypage/rewards        보상 내역
└── /users/[id]                공개 프로필 (판매자 평점, 리뷰 목록 포함)
```

## 6. 데이터 모델 요약

```
User ──────────< Template
  │                 │
  │                 └── forkedFromId → Template (복제 원본)
  │
  ├─────────── < Resume
  │                 │
  │                 └── templateId → Template (사용한 템플릿)
  │
  ├──< Purchase (targetType: TEMPLATE | RESUME)
  │
  ├──< Feedback (→ Resume)
  │
  ├──< Review (targetType: SELLER | TEMPLATE | RESUME)
  │
  ├──< RewardLog
  │
  └──< Notification
```

| 모델         | 역할                                                                                     |
| ------------ | ---------------------------------------------------------------------------------------- |
| User         | 사용자 (balance, point, bio, profileImage, avgRating, reviewCount)                       |
| Template     | 이력서 템플릿 (elements JSON, pricingType, viewCount, avgRating, reviewCount)            |
| Resume       | 완성된 이력서 (content JSON, templateId, pricingType, viewCount, avgRating, reviewCount) |
| Purchase     | 구매 기록 — 템플릿(복제 권한) / 이력서(열람 권한) 통합                                   |
| Feedback     | 이력서에 대한 피드백                                                                     |
| Review       | 평가 — 판매자/템플릿/이력서 대상, 별점(1~5) + 리뷰 텍스트, 구매 인증 뱃지 여부(isPurchased) |
| RewardLog    | 보상 적립/차감 내역                                                                      |
| Notification | 인앱 알림                                                                                |

## 7. 비즈니스 모델

### 수익 구조

```
유료 콘텐츠(템플릿/이력서) 구매 금액
├── 90% → 판매자 (balance 적립)
└── 10% → 플랫폼 수수료
```

### 결제 수단

- **실제 원화 결제** (PG사 미정)
- 포인트와 별도로 운영

### 포인트 운영

- 무료 콘텐츠 열람 보상: 플랫폼이 부담 (초기 사용자 유입/활성화 비용)
- 유료 콘텐츠 구매 수익: 수수료 10%
- **포인트 할인**: 유료 콘텐츠 원화 결제 시 보유 포인트로 할인 적용 가능
- 추후 확장: 프리미엄 에디터 기능, 광고, 기업 채용 연동 등

## 8. 개발 우선순위

### MVP (최소 기능 제품)

Phase 1~4까지 완성하면 핵심 루프가 동작합니다:

> 가입 → 템플릿 제작/탐색 → 이력서 작성 → 공유

### Phase별 사용자 가치

| Phase           | 사용자가 할 수 있는 것                                    |
| --------------- | --------------------------------------------------------- |
| 1 (인증 + 소셜) | GitHub/이메일로 가입하고 로그인                           |
| 2 (템플릿 빌더) | 노코드 빌더로 이력서 템플릿 제작                          |
| 3 (이력서 작성) | 템플릿 기반으로 이력서 완성                               |
| 4 (공유/탐색)   | 템플릿 마켓, 이력서 탐색, 공개 URL                        |
| 5 (구매 + 보상) | 유료 구매 (템플릿→복제, 이력서→조회), 보상 적립           |
| 6 (피드백)      | 이력서에 피드백 주고받기                                  |
| 7 (평가)        | 템플릿/이력서/판매자에 별점+리뷰 작성, 구매 인증 뱃지, 평점순 탐색 |
| 8 (프로필/알림) | 프로필 관리, 알림 확인, 보상 내역                         |

### MVP 이후 확장 가능성

- PDF 내보내기
- 빌더 undo/redo
- 템플릿 카테고리 세분화
- 피드백 요청 게시판
- 기업 채용 연동 (이력서 기반 스카우트)
- AI 이력서 첨삭 추천
- 이메일/푸시 알림
- 소셜 기능 (팔로우, 좋아요)
